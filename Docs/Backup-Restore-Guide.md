# Руководство по резервному копированию и восстановлению

## Структура скриптов
- `scripts/backup.sh` – создаёт дамп БД и архивирует папку `uploads`.
- `scripts/restore.sh` – восстанавливает БД и медиа-файлы из выбранной резервной копии.

## Требования
- **PostgreSQL** (локальная установка или Docker) с утилитами `pg_dump`, `psql`.
- **Bash** (на Linux/macOS) или **Git Bash** / **WSL** на Windows.
- **tar**, **gzip** для архивации.

## Создание резервной копии
Выполните из корня проекта:
```bash
./scripts/backup.sh
```

Скрипт создаст папку backups/backup_YYYYMMDD_HHMMSS/ с файлами:
 - **db_dump_YYYYMMDD_HHMMSS.sql** – дамп БД.
 - **uploads_backup_YYYYMMDD_HHMMSS.tar.gz** – архив папки uploads (если существует).
 - **backup.log** – лог выполнения.

## Восстановление из резервной копии
1. Остановите приложение.
2. Выполните (укажите путь к нужной папке бэкапа):
```bash
./scripts/restore.sh ./backups/backup_20260629_101754
```
3. Скрипт выполнит:
 - Очистку текущей базы данных.
 - Восстановление дампа .sql.
 - Распаковку архива uploads (если присутствует).
